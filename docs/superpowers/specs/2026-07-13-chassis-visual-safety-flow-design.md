# 底盘视觉、安全停车与完整任务兼容修复设计

日期：2026-07-13  
分支：`feature/mecanum3-chassis-port`

## 目标

在不改变实验室项目原有运行方式和任务控制逻辑的前提下，修复底盘移植暴露的四类问题：

1. 车体移动时四个主轮实际不转、GUI 看起来“硬走”；
2. 平面位姿驱动绕过碰撞，底盘可能进入工作台或物体；
3. 到达工位后抓取插件出现几十米的错误指尖/样件 offset；
4. MoveIt 不知道工作台几何，机械臂存在穿桌风险，完整任务无法稳定到 `DONE`。

## 不变约束

以下接口和控制结构不得因修复而改变：

- 完整启动仍为 `ros2 launch lab_cobot_bringup lab_cobot.launch.py`；
- 正式任务入口仍为 `/task/instruction`；
- 状态机仍为 `NAV_TO_PICK -> DETECT -> PICK -> NAV_TO_PLACE -> PLACE -> RETURN_HOME`；
- Nav2、MoveIt、感知和夹爪的对外接口保持不变；
- 正式底盘速度接口仍为 `/cmd_vel`；
- `mecanum_ws` 移植的轮半径、轮距、轴距、轮序、符号和逆/正解公式保持不变；
- 不切换为麦轮滚子接触动力学，不要求底盘推动物体；
- 不使用瞬移样件、关闭接触检查或关闭碰撞来掩盖任务失败。

## 已确认根因

### 主轮不转

`lab_cobot_planar_drive` 每个 Gazebo 更新周期调用模型级 `SetLinearVel` 和 `SetAngularVel`。Gazebo Classic 的模型级接口会遍历整车所有 link，覆盖 ros2_control 刚写入四个主轮关节的角速度。实测前进轮速命令为 `-2.857 rad/s`，四个轮关节 position/velocity 仍始终为零。

### 底盘穿模

插件通过 `SetWorldPose` 直接推进整车。该运动学施加方式不会自然受轮地接触约束，也不能依靠桌体碰撞阻止下一帧位姿进入障碍物。因此必须在生成下一平面位姿前做安全区约束。

### 抓取阶段失败

近期任务日志证明导航和工位精停能够完成，但抓取插件曾报告 `offset=(13.488,23.031,31.675)`，随后 MoveIt approach 失败。该量级与实际约 0.65 m 的目标距离明显矛盾，首先要消除模型级全 link 速度覆盖，再复测指尖世界位姿。

### MoveIt 缺少桌体

当前代码没有向 MoveIt planning scene 注册 station A/B 工作台。Gazebo 中存在桌体碰撞不等于 MoveIt 规划器知道桌体，因此机械臂路径可能穿过桌面。

## 方案选择

采用“原运动学 + 可视轮转动 + 双层安全区”的最小兼容方案：

- 不采用只改 waypoint 的单层方案，因为异常精停速度仍可进入桌体；
- 不采用完整轮地动力学方案，因为它会改变已确认的麦轮计算与任务稳定性；
- 上层停车逻辑保证正常任务在合适位置停车，下层插件安全区阻止异常命令造成穿模。

## 数据流

```text
/task/instruction
  -> 原 mission 状态机
  -> 原 Nav2 与工位精停
  -> /cmd_vel
  -> rover_twist_relay（原麦轮逆解）
  -> /wheel_velocity_controller/commands
       -> wheel_velocity_controller（四轮真实关节转动）
       -> lab_cobot_planar_drive（原麦轮正解）
            -> 下一平面位姿
            -> 工作台安全区过滤
            -> SetWorldPose
  -> gazebo_odom_bridge（实际位姿与差分速度）
```

## 轮子视觉修复

`lab_cobot_planar_drive` 不再调用模型级 `SetLinearVel` 和 `SetAngularVel`。插件只负责：

- 保存初始 z；
- 根据四轮命令正解并积分 x、y、yaw；
- 通过安全区检查后设置整车平面位姿；
- 保持 roll、pitch 为零；
- 在超时或异常时间跳变时停止平面积分。

四个主轮的角速度完全由 `wheel_velocity_controller` 负责。轮子 position 必须按命令持续变化；正向、横向和旋转时各轮方向必须符合源项目轮序。

由于去除模型级速度后 `/gazebo/link_states` 的瞬时 twist 可能不再代表位姿积分速度，`gazebo_odom_bridge` 改为按仿真时间对连续位姿做差，得到世界系速度，再旋转到 base frame 发布 `/odom.twist`。暂停、回退或过大 dt 时输出零速度，不跨时间跳变计算。

## 工作台安全停车

### 几何定义

world 中两张工作台保持现有几何：

```text
station A center = ( 2.0, 1.5), size = 0.8 x 0.6 m
station B center = (-2.0, 1.5), size = 0.8 x 0.6 m
```

正常抓取/放置从工作台南侧接近。以桌体 AABB 外扩安全缓冲区定义底盘禁入区域。目标为车体最前沿到桌边净距离约 `0.35 m`；计算时使用底盘简化碰撞盒投影，而不是只使用 base_link 点。

### 上层正常停车

保留 mission 的 `_navigate`、`_dock_to_station_pose` 和视觉精停调用顺序，只修改精停完成条件与速度限制：

- 距安全停车线较远时保持原控制；
- 进入减速带后按剩余距离连续降速；
- 车体外壳到桌边达到 0.35 m 时完成精停并发布零速度；
- 不再为了让样件进入更近的视觉阈值而继续推底盘；
- 机械臂从停车位完成剩余接近距离。

### 插件最后保护

插件计算下一位姿后，先计算底盘碰撞盒在世界坐标中的四角。若下一位姿会与 station A/B 的外扩安全区相交：

- 拒绝该平移分量；
- 将当前平移速度衰减到零；
- 允许明确远离安全区的命令；
- 原地旋转只有在旋转后的碰撞盒仍不进入安全区时才允许；
- 记录节流警告，避免刷屏；
- 不影响 home 和工作台以外区域。

安全区参数由 URDF 插件配置显式传入，并与 world 和 mission 测试中的桌体几何保持一致。

## MoveIt 工作台碰撞场景

新增一个职责单一的 planning scene 初始化节点，在完整 bringup 中随 MoveIt 启动阶段启动。它向 MoveIt 注册：

- `station_a_table`：中心 `(2.0,1.5,0.375)`，尺寸 `(0.8,0.6,0.75)`；
- `station_b_table`：中心 `(-2.0,1.5,0.375)`，尺寸 `(0.8,0.6,0.75)`。

碰撞物体使用 `odom`/规划参考系中与 Gazebo 一致的坐标。节点等待 planning scene 服务可用后幂等添加，重复启动不能产生重复对象。桌面对象不加入机器人自身 SRDF 禁碰撞列表。

## 机械臂可达性与抓取衔接

安全停车后按原视觉检测和抓取状态机执行。验收时先检查：

- 样件在 base_link 中的距离仍位于 UR5e 工作空间；
- approach、target、lift 三个位姿不进入桌体碰撞盒；
- 双指和样件世界位置均为有限值；
- grasp offset 恢复到配置封套允许的厘米级。

若 0.35 m 净距导致 approach 超出工作空间，只允许在保证碰撞盒不进入安全区的前提下调小净距，或调整机械臂 approach 姿态；不得让底盘继续压入桌体。

## 异常处理

- 无效轮速、命令超时：保持现有双层 watchdog 停车；
- 安全区配置无效：插件拒绝启动并给出明确错误；
- planning scene 服务未就绪：初始化节点重试，超过时限使完整启动给出错误，不静默忽略桌体；
- Nav2 还未 active：mission 不进入导航步骤，等待就绪而不是快速耗尽重试；
- MoveIt 无解：保持原失败恢复，但日志必须区分碰撞无解、不可达和控制执行失败；
- 任务失败：仍执行原停车、释放和回 home 清理路径。

## 测试与验收

### 单元和契约测试

- 锁定插件不再出现模型级 `SetLinearVel/SetAngularVel`；
- 锁定四轮命令到轮关节 position 变化的动态契约；
- 覆盖安全区 AABB、旋转碰撞盒、进入/离开/沿边运动；
- 覆盖安全距离减速和完成条件；
- 覆盖 odom 位姿差分、yaw 跨越、暂停和时间回退；
- 锁定 planning scene 两张桌体的名称、尺寸和坐标；
- 锁定原 launch、任务状态机和麦轮公式未改变。

### 动态 Gazebo 验收

1. 前进时四轮命令非零，四轮 position 连续变化，GUI 可见旋转；
2. 前后、左右、斜移、顺逆时针旋转方向正确；
3. z、roll、pitch 稳定性不低于当前版本；
4. 持续向 station A/B 发布前进命令，车体外壳在安全线前停止；
5. 停止位置到桌边净距离为 `0.35 m`，允许小范围仿真容差；
6. 底盘、机械臂、夹爪和样件均不得进入桌体；
7. `/odom` 位姿和 twist 与实测运动一致；
8. `/wheel_velocity_controller/commands` 与 `/odom` 均只有一个发布者；
9. 完整原任务至少连续两次到达 `DONE`；
10. station B 放置后样件稳定留在桌面，最终返回 home。

## 完成标准

只有同时满足以下条件才算完成：

- 原启动方式、任务入口和状态机保持一致；
- 麦轮公式、参数和轮序未改变；
- 四个主轮实际转动且视觉效果正确；
- 正常和异常命令均不能让底盘进入工作台安全区；
- MoveIt 规划场景包含两张工作台；
- 抓取 offset 为合理厘米级，机械臂不穿桌；
- 完整抓取、搬运、放置和返航达到 `DONE`；
- 相关构建、测试和动态验收通过；
- 未通过关闭碰撞、关闭触觉或瞬移样件实现成功。
