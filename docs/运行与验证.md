# 麦轮底盘移植版：运行与验证

本文适用于 `feature/mecanum3-chassis-port`。在该分支合并到 `main` 之前，请从功能 worktree 运行，不要在 `~/lab_cobot_ws` 主目录混用构建产物。

## 1. 确认目录和分支

```bash
cd ~/lab_cobot_ws/.worktrees/mecanum3-chassis-port
git status -sb
```

预期第一行包含：

```text
## feature/mecanum3-chassis-port
```

`~/lab_cobot_ws` 当前是主目录的 `main`；两个目录的 `build/`、`install/` 和 `log/` 互不通用。VS Code 也应直接打开上述 worktree，才能正确显示本分支文件状态。

## 2. 构建

每个新终端先加载 ROS 2，再在同一 worktree 构建：

```bash
cd ~/lab_cobot_ws/.worktrees/mecanum3-chassis-port
source /opt/ros/humble/setup.bash
colcon build --symlink-install
source install/setup.bash
```

修改 C++ Gazebo 插件后必须重新构建并重新启动 Gazebo；已运行的 `gzserver` 不会自动加载新插件。

## 3. 按原项目方式启动

终端 1：

```bash
cd ~/lab_cobot_ws/.worktrees/mecanum3-chassis-port
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch lab_cobot_bringup lab_cobot.launch.py
```

这仍是项目原来的一键入口，会启动 Gazebo、麦轮底盘、Nav2、MoveIt、感知、夹爪和任务节点。不要同时另开 `world.launch.py`，否则 Gazebo 端口会冲突。

终端 2：

```bash
source /opt/ros/humble/setup.bash
source ~/lab_cobot_ws/.worktrees/mecanum3-chassis-port/install/setup.bash
ros2 topic echo /task/status
```

终端 3 发送一次任务：

```bash
source /opt/ros/humble/setup.bash
source ~/lab_cobot_ws/.worktrees/mecanum3-chassis-port/install/setup.bash
ros2 topic pub --once /task/instruction std_msgs/msg/String \
  "{data: '把样件从A工位搬运到B工位，然后返回原点'}"
```

无图形界面运行：

```bash
ros2 launch lab_cobot_bringup lab_cobot.launch.py gui:=false use_rviz:=false
```

## 4. 预期现象

状态顺序应为：

```text
NAV_TO_PICK -> DETECT -> PICK -> NAV_TO_PLACE -> PLACE -> RETURN_HOME -> DONE
```

- 底盘按原任务逻辑导航，四个麦轮视觉随轮速转动；整车平面运动由与移植麦轮逆解匹配的正解积分产生。
- 底盘与工作台保持任务配置的安全距离，停车后由 UR5e 完成抓放。
- 抓取前允许出现少量 `no_finger_contact` 重试。
- 成功抓取时日志包含 `suppressed 1 attached-object collision(s)` 和 `canonical_correction=...`。
- 搬运期间仅样件自身碰撞被临时屏蔽；释放时恢复碰撞。样件最终应静止在 B 台面，中心高度约 `z=0.785 m`，不能穿过桌面。
- `/task/status` 到达 `DONE` 才表示完整任务成功。

## 6. 常见问题

### Gazebo 没启动或提示端口占用

典型错误为 `Unable to start server[bind: Address already in use]`。先检查是否已有旧进程：

```bash
pgrep -af 'gzserver|gzclient|ros2 launch'
```

先回到旧 launch 终端按 `Ctrl+C`，确认上述命令不再列出旧 Gazebo，再重新启动。不要同时运行总入口和 `lab_cobot_gazebo world.launch.py`。

### WSL localhost 代理提示

`wsl: 检测到 localhost 代理配置，但未镜像到 WSL` 是 WSL NAT 模式的代理提醒，离线启动本地 ROS/Gazebo 时通常不影响运行。只有 `apt`、`git pull` 等联网命令失败时才需要调整 Windows/WSL 代理；不要把该提示误判为 Gazebo 启动失败。

### Gazebo 中没有车或修改未生效

确认当前目录和 overlay 都来自同一 worktree：

```bash
pwd
echo "$AMENT_PREFIX_PATH" | tr ':' '\n' | head
```

修改插件后重新执行 `colcon build --symlink-install`，完全关闭旧 Gazebo，再启动。不要 source `~/lab_cobot_ws/install/setup.bash` 后运行功能 worktree。

### 结束时出现 MoveIt SIGINT/段错误日志

手动 `Ctrl+C` 结束时，MoveIt/Gazebo Classic 可能打印 shutdown 噪声。任务验收以停止前 `/task/status` 是否已到 `DONE`、样件是否稳定落桌为准。
